# Release v0.4.0 — MCP Server + Watch Mode

> Expose specs to any MCP-compatible agent and auto-verify on file changes.

This release completes Phase 3 of intake's roadmap: an MCP (Model Context Protocol) server that exposes specs as tools, resources, and prompts for any compatible AI agent; and a watch mode for continuous verification during development. Both features are optional dependencies.

## Install

```bash
pip install intake-ai-cli

# With MCP server support
pip install "intake-ai-cli[mcp]"

# With watch mode
pip install "intake-ai-cli[watch]"

# Everything
pip install "intake-ai-cli[all]"
```

Requires Python 3.12+.

## What's New

### MCP Server

Expose intake specs to any MCP-compatible AI agent (Claude Code, Claude Desktop, etc.):

```bash
# stdio transport (default)
intake mcp serve --specs-dir ./specs

# SSE transport for remote agents
intake mcp serve --transport sse --port 8080
```

#### 7 MCP Tools

| Tool | Description |
|------|-------------|
| `intake_show` | Show spec summary (requirements, tasks, checks, costs) |
| `intake_get_context` | Get project context document |
| `intake_get_tasks` | Get tasks with status filtering (pending/in_progress/done/blocked) |
| `intake_update_task` | Update task status and add notes |
| `intake_verify` | Run verification checks with optional tag filtering |
| `intake_feedback` | Analyze failures and suggest fixes |
| `intake_list_specs` | List all available specs |

#### 6 MCP Resources

Direct access to spec files via URI:

```
intake://specs/{name}/requirements  → requirements.md
intake://specs/{name}/tasks         → tasks.md
intake://specs/{name}/context       → context.md
intake://specs/{name}/acceptance    → acceptance.yaml
intake://specs/{name}/design        → design.md
intake://specs/{name}/sources       → sources.md
```

#### 2 MCP Prompts

| Prompt | Description |
|--------|-------------|
| `implement_next_task` | Load context + find next pending task + implement + verify + update status |
| `verify_and_fix` | Run verification, analyze failures, fix code, re-verify until all pass |

#### Claude Code Integration

```json
{
  "mcpServers": {
    "intake": {
      "command": "intake",
      "args": ["mcp", "serve", "--specs-dir", "./specs"]
    }
  }
}
```

### Watch Mode

Auto-verify on file changes during development:

```bash
# Watch and re-verify on changes
intake watch specs/auth/ -p .

# With tag filtering
intake watch specs/auth/ -p . -t tests -t security

# Custom debounce
intake watch specs/auth/ -p . --debounce 5.0
```

Features:
- Rust-based file watching via `watchfiles` (efficient, low CPU)
- Configurable debounce interval (default: 2 seconds)
- Ignore patterns for `.git`, `node_modules`, `__pycache__`, `.pyc`, `.intake`
- Initial verification run on start
- Rich terminal output with pass/fail summary
- Ctrl+C to stop

### Configuration

Two new config sections in `.intake.yaml`:

```yaml
mcp:
  specs_dir: ./specs
  project_dir: .
  transport: stdio    # stdio | sse
  sse_port: 8080

watch:
  debounce_seconds: 2.0
  ignore_patterns:
    - "*.pyc"
    - "__pycache__"
    - ".git"
    - "node_modules"
    - ".intake"
```

## Architecture Changes

### New Modules

| Module | Purpose |
|--------|---------|
| `mcp/__init__.py` | MCPError exception |
| `mcp/server.py` | Server creation, stdio and SSE transports |
| `mcp/tools.py` | 7 tool definitions and handlers |
| `mcp/resources.py` | Resource listing and reading via URI |
| `mcp/prompts.py` | 2 prompt templates (implement_next_task, verify_and_fix) |
| `watch/__init__.py` | WatchError exception |
| `watch/watcher.py` | SpecWatcher with run/run_once, debouncing, ignore filtering |

### Modified Modules

| Module | Change |
|--------|--------|
| `config/schema.py` | Added MCPConfig and WatchConfig models |
| `cli.py` | Added `mcp serve` and `watch` commands |
| `pyproject.toml` | New optional deps: `[mcp]`, `[watch]`, `[all]` |

### New Optional Dependencies

```toml
[project.optional-dependencies]
mcp = ["mcp>=1.0"]
watch = ["watchfiles>=0.20"]
all = ["intake-ai-cli[connectors,mcp,watch]"]
```

### Technical Decisions

- Lazy imports for `mcp`, `watchfiles`, `starlette`, `uvicorn` — never fail at import time
- `fnmatch` for ignore patterns, matching against each path component
- `SpecWatcher.run_once()` for testability without blocking
- MAX_SECTION_LENGTH = 3000 chars per spec file in MCP tools
- MAX_CHANGED_FILES_DISPLAY = 5 in watch output
