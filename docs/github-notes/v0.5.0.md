## What's New

### GitHub Actions Composite Action

A ready-to-use action that installs intake, runs verification, and uploads reports as artifacts:

```yaml
- uses: Diego303/intake-cli/action@main
  with:
    spec-dir: specs/mi-feature/
```

#### Inputs

| Input | Default | Description |
|-------|---------|-------------|
| `spec-dir` | *(required)* | Path to the spec directory |
| `project-dir` | `.` | Project directory to verify against |
| `report-format` | `junit` | Report format: `terminal`, `json`, `junit` |
| `report-output` | `intake-verify-report.xml` | Report file path |
| `tags` | `""` | Comma-separated check tags |
| `fail-fast` | `false` | Stop at first failure |
| `python-version` | `3.12` | Python version |
| `intake-version` | `latest` | intake version constraint |

#### Outputs

| Output | Description |
|--------|-------------|
| `result` | `pass`, `fail`, or `error` |
| `total-checks` | Total checks executed |
| `passed-checks` | Checks passed |
| `failed-checks` | Checks failed |
| `report-path` | Path to report file |

### CI Pipeline

`.github/workflows/ci.yml` with 4 jobs:

| Job | What It Does |
|-----|--------------|
| **lint** | `ruff check` + `ruff format --check` |
| **typecheck** | `mypy --strict src/` |
| **test** | `pytest` on Python 3.12 + 3.13, with coverage |
| **build** | `python -m build` to verify package builds |

Concurrency control ensures only one CI run per branch.

### mypy --strict Compliance

All `type: ignore` comments now use specific error codes. Optional dependency modules have proper mypy overrides:

```toml
[[tool.mypy.overrides]]
module = [
    "mcp.*", "atlassian.*", "github.*",
    "uvicorn", "starlette.*", "watchfiles"
]
ignore_missing_imports = true
```

### Error Handling Hardening

- **Feedback command**: catches `FileNotFoundError` and `json.JSONDecodeError` specifically instead of generic exceptions
- **Connector temp files**: all 3 connectors (Jira, Confluence, GitHub) catch `OSError` when writing temp files and wrap in `ConnectorError` with actionable messages
- **Connector fetch**: `_fetch_connector_source` in CLI handles `TimeoutError` and `OSError` separately

### Technical Decisions

| # | Decision | Rationale |
|---|----------|-----------|
| 35 | Composite action over Docker action | Faster startup (no container build), simpler maintenance, transparent steps |
| 36 | mypy overrides per-module instead of inline `type: ignore[import]` | Cleaner code, single source of truth for ignored imports |
| 37 | Specific error codes in `type: ignore` comments | Catches real errors that would be hidden by bare `type: ignore` |
| 38 | Matrix test on 3.12 + 3.13 | Ensures forward compatibility without increasing CI time |
